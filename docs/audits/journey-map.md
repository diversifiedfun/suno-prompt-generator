# Journey Map — Suno Prompt Studio

> Generated by `/journey-audit-init-extension` on 2026-07-08. Authoritative input for the weekly `/journey-audit` routine.
> Regenerate by re-running `/journey-audit-init-extension` in this repo when surfaces/tabs/host couplings change materially.

- **Repo:** diversifiedfun/suno-prompt-generator
- **Manifest version:** 3
- **Surfaces:** side panel only (`extension/sidepanel.html`, opened via the toolbar action icon). No popup, no options page.
- **Background:** service_worker `extension/background.js` (module; opens the side panel + context menus).
- **Content-script host matches:** `https://suno.com/*` → `extension/suno-fill.js` (fills Style/Lyrics/Title/Exclude via the React native-setter). Slider control is a SEPARATE seam: `extension/suno-slider-driver.js` injected from the side panel via `chrome.debugger` (CDP trusted drags) — the two together are the highest-risk coupling (third-party DOM + CDP).
- **Message seams:**
  - side panel → `chrome.tabs.sendMessage(tabId, {type:"suno-fill", ...})` → `sunoFillHandler` listener in `suno-fill.js`; on a stale tab, side panel falls back to `chrome.scripting.executeScript` re-inject + retry.
  - side panel → `chrome.debugger.attach` + `Input.dispatchMouseEvent` (in `suno-slider-driver.js`) to move Weirdness / Style-Influence sliders, then re-reads `aria-valuenow` to verify.
  - Listener install uses a **handler-ref swap** (not a one-shot boolean) to avoid the dead-listener-after-reload bug (`suno-fill.js:97-105`) — keep it that way.
- **Credentials model:** user's own Anthropic API key, entered in the Settings tab, stored in `chrome.storage.local` under `settings`. ⚠️ Stored **untrimmed** (`storage.js` `setSettings`) → a pasted key with trailing whitespace silently 401s. No get-a-key link at the field; no format check / test.
- **Storage model:** `chrome.storage.local`, per-key schema — `prompt_<id>` (Library), `vibe_<id>` (Saved vibes), `genhist_<id>` (Recent generations, capped 500, starred exempt), `set_<id>`, `album_<id>`, plus `settings` and `uiState`. Cross-session restore via `uiState` (active tab + Vibe/Build inputs + last result), gated by a `booted` flag so startup init can't clobber restore. Known gap: `loadGeneration` (history "Load") updates the DOM but NOT `uiState` → lost on reopen.
- **Permission trust surface:** `permissions: [contextMenus, storage, sidePanel, tabs, scripting, debugger]`, `host_permissions: [api.anthropic.com, suno.com]`. ⚠️ **`debugger`** is the scary one — Chrome shows a "started debugging this browser" banner on every slider-driving paste, with NO in-UI forewarning (trust risk + Chrome Web Store review risk). `tabs` + `scripting` are needed for the Suno handoff. No `<all_urls>` (good — host-scoped).

## Journeys (authoritative list)

1. **Install → first value** — action icon → side panel opens on `sidepanel.html` → Vibe tab. First-run banner explains setup. Works OFFLINE without a key (curated library) — good, value before credentials.
   - Fragile handoffs: first-run banner copy says "free-to-start" but the Anthropic key needs prepaid credits (contradicts next screen); the offline path is real but under-explained.
2. **Permissions & trust** — install prompt from the manifest; runtime `debugger` banner on first slider-paste.
   - Fragile: the "started debugging this browser" banner appears with zero forewarning → reads as malware. `debugger` is a store-review risk (flagged in `store-submission` memory).
3. **Credentials / API-key onboarding** — Settings tab → paste `sk-ant-…` → Save.
   - Fragile: untrimmed key → silent 401 that reads as a lie; "where to get a key" is buried in a collapsed `<details>` below Save; no validation/test on save.
4. **Core action → host-page handoff (THE seam)** — Vibe/Set/Album tab generates → "Paste → Suno" → `suno-fill.js` fills the Suno DOM → `suno-slider-driver.js` drives sliders.
   - Fragile: Suno tab not open; **not logged in** (fill can "succeed" against the logged-out landing widget → false success); Suno in **Simple mode** (Style box absent → "switch to Custom mode" only surfaces AS the failure, never forewarned); stale selectors on a Suno redesign (fails loudly — good, single-file `suno-fill.js`); captcha wall after rapid Creates (unsolvable — documented in `nova-reign` memory); the slider CDP drive verifies `aria-valuenow` and falls back to showing numbers (honest).
5. **Persistence & return** — `chrome.storage.local` + `uiState` restore.
   - Fragile: `loadGeneration` doesn't persist (lost on reopen); THREE overlapping stores (Library / History / Saved Vibes) with inverted affordances; Library can only Copy, not Paste → Suno.
6. **Discovery & navigation** — tabs: Vibe (generate) · Build · Set · Album · Library · Settings.
   - Fragile: "Build" verb overloaded across 3 tabs; Set/Album/Vibe choice unexplained up front; history + saved-vibes are collapsed accordions styled like help text → users don't recognize them as their own data.
7. **Help & error recovery** — inline `.warn` blocks + paste status strings; per-tab "❔ How this works".
   - Fragile: one bare `alert()` on empty generate input; save-failure `catch` is console-only (silent dead click); no `aria-live` on status regions.
8. **Offboarding & data hygiene** — "Clear all" in history (keeps starred, fixed in v1.6.2); delete per Library/vibe/set/album; no global "clear everything" or key-revoke affordance.
   - Fragile: no one-shot data wipe or key-remove; uninstall leaves `chrome.storage.local` (acceptable for a local tool, but undocumented).
9. **Share / export** — Copy per-field + "Copy whole set/album". No export file, no shareable artifact.
   - Fragile: outputs are copy-only; no way to hand a set to someone else as a file.
